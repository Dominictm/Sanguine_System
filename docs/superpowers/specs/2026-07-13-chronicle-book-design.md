# Спека — компиляция хроники в «книгу»

Дата: 2026-07-13
Фаза 7 дорожной карты `docs/superpowers/plans/2026-07-13-development-roadmap.md`.

## Задача

Хроника одним читабельным HTML-документом (печать → PDF средствами
браузера): спина и финал хроники, события по порядку, финалы модулей.

## Архитектура: сборка на клиенте

Серверного MD→HTML рендерера в проекте нет, а у клиента есть
`mdToHtmlBlock`. Книга собирается **на клиенте** и открывается новой
вкладкой (`window.open` → `document.write`), документ самодостаточен —
собственный инлайн-CSS (тёмная тема проекта + `@media print` с светлой
печатной версией), без ссылок на арт (относительные пути из markdown в
отдельном документе всё равно битые — вычищаются).

## Сервер (минимум)

`web/routes/chronicles.js`: `GET /api/chronicles/:slug/book-data` →
`{ display, chronicleMd, modules: [{name, title, finale}] }` — читает
`chronicles/<slug>/chronicle.md` (raw) и `finale.md` каждого модуля
(имена — существующей логикой списка модулей). 404 если хроники нет.
Только чтение. События клиент берёт существующим
`GET /api/chronicles/:slug/events`.

## Фронтенд

`web/public/scripts/modules.js`: кнопка «📕 Книга» в шапке модалки хроники
(рядом с «🕰️ В хронологию»); обработчик: параллельно `book-data` + `events`,
рендер разделов: титул (название, город, дата сборки) → chronicle.md
(спина+финал) → события (дата, заголовок, локация, текст — как chron-md) →
финалы модулей (по одному разделу). Markdown → `mdToHtmlBlock`; ссылки
`[text](path.md)` перед рендером упрощаются в `text` (в книге они мертвы).

## Тесты / приёмка

- Integration: `book-data` существующей хроники → display, chronicleMd
  непуст, modules с finale у закрытого модуля; несуществующая → 404.
- CDP: кнопка открывает вкладку, документ содержит титул/спину/финалы,
  консоль чистая.
