# Звуковая библиотека: категории и аудио-пресеты — дизайн

**Дата:** 2026-07-10
**Статус:** утверждено пользователем, готово к написанию implementation plan
**Дополняет:** `docs/superpowers/specs/2026-07-10-audio-library-design.md` (базовая
реализация — загрузка/плеер/громкость/зацикливание, уже реализована в
`web/routes/audio.js` + `web/public/audio-library.js`).

## Цель

Две доработки уже работающей звуковой библиотеки (саундборда):

1. Каждый звук получает категорию — «Фоновая музыка» или «Аудио эффекты»
   (взаимоисключающий выбор, обязателен при загрузке, меняется одним кликом
   на карточке).
2. Страница делится на 3 колонки: Фоновая музыка / Эффекты / Аудио пресеты.
   Пресет — сохранённый набор звуков с индивидуальной громкостью каждого,
   запускаемых одновременно одной кнопкой; опционально привязан к локации
   (на карточке пресета — краткое название и картинка локации).

## Модель данных

### Категория трека (`cities/audio/index.json`)

Новое поле `category: 'music' | 'effect'` в каждой записи (существующие поля
без изменений: `id, ext, filename, title, volume, loop, createdAt`). Пример:

```json
{ "id": "...", "ext": "mp3", "filename": "storm.mp3", "title": "Гроза за окном",
  "volume": 1, "loop": true, "category": "music", "createdAt": "..." }
```

Обязателен при создании — `POST /api/audio` отклоняет запрос без валидного
`category` (400). Существующие записи, загруженные до этой доработки, поля не
имеют — при рендере трактуются как `'effect'` (`t.category !== 'music'`),
без миграции файла на диске; при первом ручном клике по переключателю
категории на карточке поле проставляется явно.

### Пресеты (`cities/audio/presets.json`, новый файл)

Отдельный манифест, не смешивается со списком треков — самостоятельная
сущность с собственным CRUD, как и `index.json` для треков.

```json
[
  {
    "id": "uuid",
    "name": "Погоня в переулке",
    "locationSlug": "rue-morgue",
    "tracks": [
      { "trackId": "uuid-трека-1", "volume": 0.6 },
      { "trackId": "uuid-трека-2", "volume": 1.0 }
    ],
    "createdAt": "2026-07-10T12:00:00.000Z"
  }
]
```

- `locationSlug` — опциональная ссылка на локацию текущего города по `slug`
  (может быть `null`). Название и картинка локации **не дублируются** в
  пресете — при рендере подтягиваются живьём из `GET /api/locations`
  (`title`, `imageUrl`) по `locationSlug`, чтобы переименование/добавление
  картинки локации не протухало данные пресета.
- `tracks[].volume` — громкость конкретного трека **внутри этого пресета**,
  независимая от общей громкости трека в библиотеке (пользователь явно
  выбрал раздельные значения — в одном пресете эффект громче, в другом тише).
- Если трек, на который ссылается пресет, удалён из библиотеки — при запуске
  пресета такая ссылка тихо пропускается (остальные треки пресета всё равно
  играют), без ошибки на весь пресет.

## API

Новые/изменённые эндпоинты в `web/routes/audio.js`:

- `POST /api/audio` — тело дополняется обязательным `category` (`'music'` |
  `'effect'`); отсутствие или неверное значение → 400.
- `PUT /api/audio/:id` — дополнительно принимает `category` (переключатель
  на карточке).
- `GET /api/audio/presets` — список пресетов, с подстановкой `locationTitle`
  / `locationImageUrl` (или `null`, если `locationSlug` не задан либо
  локация не найдена — не 404, тихий null) и `tracks[].title`/`url` каждого
  входящего трека (подтянуто из `index.json` по `trackId`, для рендера
  карточки без второго запроса).
- `POST /api/audio/presets` — тело `{ name, locationSlug, tracks: [{trackId, volume}] }`;
  `name` обязателен и непуст, `tracks` — непустой массив существующих
  `trackId`.
- `PUT /api/audio/presets/:id` — точечное обновление `name`/`locationSlug`/
  `tracks` (в т.ч. изменение громкости одного трека внутри пресета — весь
  массив `tracks` передаётся целиком, как и для остальных PUT в проекте).
- `DELETE /api/audio/presets/:id` — удаляет запись из `presets.json` (сами
  аудиофайлы не трогает — пресет не владеет файлами, только ссылается).

## Фронтенд

### Страница — 3 колонки

`#page-audio-library` вместо одной сетки `.lib-cards` — три колонки:
«Фоновая музыка» (`t.category === 'music'`), «Эффекты» (остальные), «Аудио
пресеты». Кнопка «+ Загрузить звук» и глобальная «⏹ Остановить всё» — как
сейчас, общие на странице. Новая кнопка «+ Создать пресет» — открывает
отдельную модалку.

### Форма загрузки/редактирования звука

Модалка загрузки получает 2 взаимоисключающих radio-кнопки «Фоновая музыка» /
«Аудио эффекты» (обязательный выбор, без значения по умолчанию — форма не
даёт отправить без выбора). На карточке трека — новая кнопка-переключатель
категории (аналогично уже реализованной кнопке зацикливания 🔁): один клик
мгновенно переключает категорию, карточка перерисовывается в соответствующую
колонку, изменение сохраняется через `PUT`.

### Карточка пресета

Показывает:
- название пресета;
- если `locationSlug` задан — краткое название локации и её картинка
  (`imageUrl`), иначе — ничего (пресет без локации, это нормальный случай);
- список входящих звуков, каждый со своим ползунком громкости (та же
  UI-логика, что и громкость обычного трека: живо меняет `audioEl.volume`
  во время воспроизведения и сохраняет через debounced `PUT
  /api/audio/presets/:id`);
- кнопку запуска/остановки пресета.

### Запуск пресета

Клик по кнопке запуска: сначала вызывается тот же код, что и у «Остановить
всё» (останавливает и сбрасывает позицию всех треков на странице — не
накладывать пресет на уже играющий фон/эффект другой сцены), затем
запускаются все треки текущего пресета с их `tracks[].volume`. Кнопка
переключается в состояние «⏹ Остановить [название]» — своя кнопка остановки
именно у этого пресета (не только через глобальную «Остановить всё»),
неактивные пресеты остаются в состоянии «▶ Запустить». Поскольку запуск
любого пресета останавливает всё остальное, одновременно активным может быть
только один пресет — простое булево состояние на карточке, без отслеживания
пересечений.

### Модалка создания/редактирования пресета

Поле «Название» (обязательное), чек-лист всех треков библиотеки (музыка и
эффекты вперемешку — пресет может сочетать оба), рядом с отмеченным треком —
ползунок громкости (по умолчанию равен текущей громкости трека из
библиотеки, редактируется независимо для этого пресета), выпадающий список
локаций текущего города (`GET /api/locations`, опция «Без локации» первой).

## Ошибки и крайние случаи

- `POST /api/audio` без `category` или с недопустимым значением → 400.
- `POST /api/audio/presets` с пустым `tracks` или без `name` → 400.
- Запуск пресета, где часть `trackId` уже не существует в библиотеке (трек
  удалили после создания пресета) — воспроизводятся только оставшиеся треки,
  без ошибки на весь пресет.
- Пустая колонка (нет треков нужной категории / нет пресетов) — тот же
  паттерн пустого состояния, что уже есть («Библиотека пуста…»).

## Тестирование

- Юнит-тесты роутера: `POST /api/audio` без category → 400; с `category`
  → сохраняется и возвращается; `PUT` меняет category; `GET/POST/PUT/DELETE
  /api/audio/presets` — создание, точечное обновление громкости трека внутри
  пресета, удаление, 404 на несуществующий id, устойчивость к ссылке на уже
  удалённый `trackId`.
- Ручная проверка в браузере (`run-sanguine-web`): смена категории на
  карточке двигает её между колонками; создание пресета с треком и без
  локации/с локацией; запуск пресета останавливает прочие треки и запускает
  свои с заданной громкостью; своя кнопка остановки пресета; удаление трека,
  на который ссылается пресет, не ломает воспроизведение остальных.

## Вне объёма

- Порядок/сортировка колонок, перетаскивание карточек между колонками мышью
  (переключение — только кнопкой на карточке).
- Ограничение на число треков в пресете.
- Пресеты, общие между городами — привязка к локации подразумевает, что
  пресет живёт в контексте текущего города (сама библиотека звуков и без
  того город-нейтральна, но `locationSlug` резолвится в рамках активного
  города на момент рендера).
