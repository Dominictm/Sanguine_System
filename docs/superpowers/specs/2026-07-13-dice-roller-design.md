Дата: 2026-07-13
Фаза 3 дорожной карты `docs/superpowers/plans/2026-07-13-development-roadmap.md`.

## Правила броска (решения пользователя)

Пул d10 против сложности (2–10, по умолчанию 6).

- **Успех** — кубик ≥ сложности.
- **«10» — 10-again (хоусрул стола, выбран пользователем):** каждая десятка
  даёт успех **и** перебрасывается; перебросы тоже могут дать успех и снова
  взорваться. Единицы на перебросах **не** вычитают успехи и **не** влияют
  на ботч (считаются только единицы исходного пула). Предохранитель — не
  более 50 перебросов за бросок.
- **Ботч — канон V20:** единицы исходного пула вычитают успехи;
  ботч = 0 успехов **до вычета** И хотя бы одна единица.
- Итог: `botch` | `failure` (чистых успехов ≤ 0) | `success`.

## Реализация

**Новый файл `web/public/scripts/dice.js`** (classic script, подключается в
`index.html` перед `scripts.js`):

- Чистая функция `rollV20Pool({ pool, difficulty, rng })` →
  `{ dice, rerolls, successes, ones, net, result }`; `rng` инъектируется
  (по умолчанию `Math.random`) — детерминированные unit-тесты.
  В конце файла `if (typeof module !== 'undefined') module.exports = …` —
  тот же файл require-ится тестами Node без eval-хаков.
- **Плавающий виджет**: кнопка 🎲 (fixed, правый нижний угол, существующий
  z-index-токен), панель: пул (1–20), сложность (2–10, default 6),
  «Бросить», результат (кубики чипами, десятки/единицы подсвечены, чистые
  успехи или «БОТЧ»), история последних 10 бросков (в памяти сессии).
- **Интеграция с листом**: если открыт V20-лист (`_v20Model` непуст) — в
  панели два селекта «Атрибут» (9 фиксированных, значения из модели) и
  «Способность» (из `abilities.*` модели, только с именем); выбор
  подставляет сумму точек в пул. Разметка листа не трогается.
- Лог в сессию — **вне рамок** (уедет в фазу 4 «Экран сессии»).
- CSS — токены `:root`, touch ≥44px.

## Тесты / приёмка

- Unit (require `dice.js`): фиксированный rng — успехи/единицы/вычет;
  ботч (0 успехов + единица); не-ботч при вычете в ноль от единиц при
  наличии успеха; 10-again цепочка (десятка → переброс → успех);
  единица на перебросе не вычитает.
- CDP: кнопка видна на любой странице; бросок 5к6 показывает результат и
  пишет историю; при открытом листе селекты подставляют пул; консоль чистая.
