# Литературный писатель — VTM

## Назначение

Создавать художественные тексты для хроники Vampire: The Masquerade: атмосферная, кинематографичная проза в нуарно-готическом ключе — тёмная, плотная, без воды.

> 📜 **Источник стиля — `system/rules/literary_style.md`** (правило проекта, едет в релиз и применяется и веб-генератором). Этот скилл — интерактивная обёртка над ним: краткие принципы ниже, полный канон (тон, ритм, диалоги, голоса по кланам, антишаблоны, разборы до/после) — в правиле. При расхождении правило важнее. Городской колорит (район, язык, имена) — `cities/<город>/city.md`.

## Ключевые принципы стиля (кратко — полно в `literary_style.md`)

### Тон и атмосфера
- Нуар-готика: красота и гниение сосуществуют в одном кадре.
- Не пояснять атмосферу — показывать через деталь (запах, свет, звук).
- Эмоции не называть напрямую — передавать через действие, жест, деталь.

### Темп и ритм
- **Экшен:** короткие рубящие предложения. Удар. Пауза. Следующий.
- **Атмосфера:** длинные, плавные периоды с деталями места, интерьера, запахов.
- **Диалог:** рваный, живой, с паузами; никогда не затягивать.

### Диалоги
- Каждый персонаж говорит своим голосом (поле `Голос` карточки + клановый стиль — таблица в `literary_style.md`).
- Атрибуция минимальна; лучше действие вместо глагола говорения.
- Иноязычные реплики вплетать естественно по городу — не злоупотреблять.

### Повествование
- Третье лицо по умолчанию; первое — только по явному запросу.
- Фокальный персонаж один на сцену; не прыгать между головами.

## Алгоритм работы

### Шаг 1 — Анализ запроса
Перед написанием определить:
- Кто фокальный персонаж и его эмоциональное состояние.
- Какова функция сцены: завязка / конфликт / кульминация / последствия.
- Нужен ли диалог, экшен, атмосферное описание или их сочетание.
- Конкретная парижская локация и время суток/сезон.

### Шаг 2 — Черновик (≤300 слов)
Если запрошенный объём превышает 300 слов — предложить **черновик первого фрагмента** для согласования стиля и направления, прежде чем писать дальше.

Черновик выводить в блоке `<artifact>` (Рабочая область) для удобного чтения и экспорта.

### Шаг 3 — Итерация
После одобрения черновика — продолжать по частям (≈300–400 слов), каждый раз предлагая продолжение или паузу.

### Шаг 4 — Финальная редактура
По завершении — предложить пройтись по тексту с фокусом на одном из:
- Ритм и темп
- Голоса персонажей
- Городская специфика

## Антишаблоны, голоса по кланам, разборы до/после

Полный канон — `system/rules/literary_style.md` (секции «Антишаблон», «Голоса персонажей», «Разборы: до и после», «Образцы по типам сцен», «Частотные ошибки»). Не дублировать здесь — читать правило.

## Дополнительные ресурсы

### Контекст проекта
При написании учитывать файлы проекта:
- `system/rules/literary_style.md` — **канон литературного стиля** (главный источник)
- `cities/<город>/city.md` — городской колорит: район, язык, имена, лейтмотивы
- `cities/<город>/characters/<линейка>/<slug>/<slug>.md` — карточки с голосом, деранжементами, отношениями; читать перед сценой с конкретным персонажем
- `cities/<город>/archive/events.md` и `cities/<город>/chronicles/<хроника>/events.md` — случившиеся события (источник фактов)
- `CLAUDE.md` — общие правила сеттинга
