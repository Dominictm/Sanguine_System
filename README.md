# VTM Париж 2010 — Проект Хроники

Vampire: The Masquerade 20th Anniversary Edition. Хроника разворачивается в Париже, 2010 год. Проект ведётся в связке с Claude Code как Рассказчиком и сценаристом.

## Быстрый старт

| Задача | Команда / Действие |
|---|---|
| **Новый НПС (канонический)** | `.\tools\new_npc.ps1 -Name "Имя" -Type vampire` |
| **Новый НПС (модульный)** | Claude создаёт карточку по Шаблону Г в `modules/.../нпс/` |
| **Продвинуть НПС в канон** | `.\tools\promote_npc.ps1 -ModuleName "..." -NpcName "Имя"` |
| **Новый модуль** | `.\tools\new_module.ps1` (интерактивный визард) |
| **Новая локация** | `.\tools\new_location.ps1 -Name "..." -District "Район_NN" -Zone neutral` |
| **Дневник персонажа** | `.\tools\add_diary.ps1 -CharacterName "Имя" -Period "2010-11"` |
| **Записать сессию** | Описать события → Claude заполнит `Stories_of_Paris.md` |
| **Написать сцену** | `/literary-writer` + описание запроса |
| **Проверить ссылки** | `.\tools\validate_links.ps1` |
| **Статусы персонажей** | `.\tools\status.ps1` |
| **Поиск по проекту** | `.\tools\search.ps1 "запрос"` |
| **Открытые нити** | `rules/open_threads.md` |
| **Чеклист** | `rules/CHECKLIST.md` |

## I. Персонажи

Полный протокол — `rules/npcs_city.md`. Чеклист по шагам — `rules/CHECKLIST.md` → секция 1.

### Канонические vs. модульные НПС

| Тип | Где хранится | Шаблон | Как создать |
|---|---|---|---|
| **Канонический** | `characters/[линейка]/[Имя]/` | А (вампир) / Б (смертный) / В (фея) | `.\tools\new_npc.ps1` |
| **Модульный** | `modules/[модуль]/нпс/[Имя]/` | Г (упрощённый) | Claude по запросу |

**Критерии продвижения в канон:** выжил в модуле + упомянут в `финал.md` + появился в 2+ модулях, *или* явный запрос пользователя. Инструмент: `.\tools\promote_npc.ps1`.

Сводный справочник: `characters/characters_ALL.md`.

### V20-лист механик

Создаётся в отдельном файле `[Имя]-лист.md` рядом с карточкой. Правила и шаблон — `rules/character_sheet_v20.md`. Лист создаётся по запросу для ПК и важных НПС-бойцов.

### Переименование персонажа

Требует правок в 10+ файлах. Всегда использовать чеклист `rules/CHECKLIST.md` → секция 3, затем запускать `.\tools\validate_links.ps1`.

## II. Модули

Полный протокол — `rules/module_rules.md`. Чеклист — `rules/CHECKLIST.md` → секция 2.

### Создать новый модуль

```powershell
.\tools\new_module.ps1
```

Интерактивный визард: спрашивает про связь с хроникой, список НПС и локаций. Создаёт папку с файлами-заготовками.

После запуска:
1. Заполнить `[описание].md` и `сценарий.md`
2. Заполнить `нпс.md` и создать карточки модульных НПС в `нпс/`
3. Добавить запись в `Stories_of_Paris.md` по протоколу из `rules/chronicle_paris.md`
4. Обновить `rules/open_threads.md`
5. Создать дневники участников через `.\tools\add_diary.ps1`

### Структура модуля

```
modules/ноябрь_2010_кошки_и_мышки/
├── кошки_и_мышки.md   # Основной файл: параметры, связь с хроникой
├── сценарий.md         # Сцены, завязка, кульминация, варианты финала
├── нпс.md              # ПК / Каноничные НПС / Модульные НПС (ссылки)
├── финал.md            # Литературный финал (создаётся после сессии)
└── нпс/                # Папки модульных НПС
    └── [Имя]/
        └── [Имя].md   # Карточка Шаблон Г
```

## III. Хроника

Все события — `Stories_of_Paris.md`. Протокол добавления — `rules/chronicle_paris.md`. Открытые нити — `rules/open_threads.md`.

Хроника обновляется после каждой сессии. Протокол (6 шагов: проверка хронологии → валидация НПС → шаблон записи → обновление состояния мира → дневники → подтверждение) — в самом файле хроники.

---

## IV. Дневники

Каждый ПК и важный НПС может иметь личный дневник. Правила — `rules/diary_rules.md`.

```powershell
# Новый месяц:
.\tools\add_diary.ps1 -CharacterName "Эмилия" -Period "2010-11"

# Ретроспектива:
.\tools\add_diary.ps1 -CharacterName "Верене де Кюстин" -Period retrospective

# С привязкой к модулю:
.\tools\add_diary.ps1 -CharacterName "Джек" -Period "2010-11" -ModuleRef "ноябрь_2010_кошки_и_мышки"
```

Файлы хранятся в `characters/[линейка]/[Имя]/Journal_[Имя]/`.

## V. Локации

Карточки локаций — `locations/`. Правила и промты — `rules/portret.md`.

```powershell
# Парижская локация:
.\tools\new_location.ps1 -Name "Склад в Ла-Курнёв" -District "Сен-Дени_93" -Zone dangerous

# Локация вне Парижа:
.\tools\new_location.ps1 -Name "Вилла в Лионе" -District Другие -Module "февраль_2011_лион" -Zone neutral
```

Доступные районы: `Лувр_01` · `Тампль_03` · `Отель-де-Виль_04` · `Пантеон_05` · `Сен-Жермен_06` · `Антрепо_10` · `Опера_09` · `Монпарнас_14` · `Монмартр_18` · `Менильмонтан_20` · `Ла-Дефанс_BD` · `Сен-Дени_93` · `Метро_Подземный`

## VI. Художественные тексты

Стиль: нуар-готика. Показывать, не называть. Каждый персонаж говорит своим голосом.

Для написания сцен — skill `/literary-writer`. Детальный гид по стилю — `.claude/skills/literary-writer/references/style-guide.md`.

---

## VII. Визуальные референсы

Изображения — в `img-personj/`. Таблица сопоставления персонаж ↔ файл — `rules/npc_image_mapping.md`. Правила промтов — `rules/portret.md`.

При добавлении нового изображения: обновить карточку персонажа → добавить строку в `npc_image_mapping.md`.

## VIII. Инструменты

| Скрипт | Назначение | Что автоматизирует |
|---|---|---|
| `new_npc.ps1` | Новый канонический НПС | Папка + шаблон + `characters_ALL.md` + `npc_image_mapping.md` |
| `promote_npc.ps1` | НПС модуля → канон | Перемещение, обновление индексов, замена ссылок в модулях |
| `new_module.ps1` | Новый модуль | Папка + заготовки всех файлов, связь с хроникой |
| `new_location.ps1` | Новая карточка локации | Папка + шаблон + базовый промт (11 обязательных элементов) |
| `add_diary.ps1` | Дневниковая запись | Поиск персонажа, папка `Journal_`, шаблон записи, append-режим |
| `validate_links.ps1` | Проверка ссылок | Проход по всем `.md`, отчёт о битых путях |
| `status.ps1` | Дашборд статусов | Список персонажей с полем Статус |
| `search.ps1` | Поиск по проекту | Grep по всем `.md` файлам |
| `backup.ps1` | Резервное копирование | Архив проекта |

### validate_links.ps1 — флаги

```powershell
# Из корня проекта:
.\tools\validate_links.ps1

# С фильтром по папке:
.\tools\validate_links.ps1 -Filter "characters"

# Включая rules/ (шаблонные пути):
.\tools\validate_links.ps1 -IncludeRules
```

### Что остаётся Claude (не автоматизируется)

| Скрипт | Требует Claude |
|---|---|
| `new_npc.ps1` | Биография, голос, промт изображения, цвет фона |
| `promote_npc.ps1` | Расширить биографию, поколение, сир, иерархия, дисциплины |
| `new_location.ps1` | Атмосфера, сенсорная палитра, крючки, уточнение промта |
| `add_diary.ps1` | Текст записи, зеркальные ссылки, скрытые детали |
