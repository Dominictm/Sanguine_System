# Звуковая библиотека (саундборд) — дизайн

**Дата:** 2026-07-10
**Статус:** утверждено пользователем, готово к написанию implementation plan

## Цель

Новый раздел в навигации — «Звуковая библиотека»: общая для всех городов
(город-нейтральная) коллекция аудиодорожек, которую Рассказчик использует как
саундборд во время сессии — проигрывает несколько звуков одновременно
(эмбиент + акцентный звук поверх сцены), а не как одиночный медиаплеер.

## Хранение данных

- Папка `cities/audio/` — на уровне города (`cities/<city>/`), но НЕ привязана
  к конкретному городу; это ровно та же семантика, что у `system/` (глобальные
  данные одни на всё), только физически лежит под `cities/` по прямому
  указанию пользователя.
- **Вся `cities/audio/` добавляется в `.gitignore` целиком.** Аудиофайлы и
  манифест не коммитятся — каждый пользователь собирает свою библиотеку
  локально, в репозиторий это не попадает (чтобы не раздувать проект весом
  аудио). Как следствие, `tools/build_release.js` не нужно модифицировать:
  раздел в git никогда не появляется, поэтому конфликта с вычисткой
  `cities/<город>/` при сборке релиза нет.
- `cities/audio/index.json` — манифест-массив:
  ```json
  [
    { "id": "3f9c1e20-...", "filename": "storm_ambience.mp3", "title": "Гроза за окном", "volume": 1.0, "createdAt": "2026-07-10T12:00:00.000Z" }
  ]
  ```
  - `id` — uuid, файл на диске называется `<id>.<ext>` (не исходное имя —
    снимает проблемы с юникодом/коллизиями/спецсимволами в названии).
  - `filename` — исходное имя файла (для информации, не используется как путь).
  - `title` — отображаемое название (редактируемое отдельно от файла).
  - `volume` — громкость по умолчанию, 0.0–1.0.

## API (`web/routes/audio.js`, новый роутер)

- `GET /api/audio` — читает `index.json` (создаёт пустой список, если файла
  или папки ещё нет), отдаёт список с добавленным `url: /audio-files/<id>.<ext>`.
- `POST /api/audio` — тело `{ title, filename, mimetype, data }`, где `data` —
  base64 без `data:...;base64,` префикса (клиент режет сам). Валидация:
  - `mimetype` ∈ `{audio/mpeg, audio/ogg, audio/wav, audio/x-wav}` — иначе 400.
  - декодированный размер ≤ 20 МБ — иначе 400.
  - `title` непустой (обрезанный `trim()`) — иначе 400.
  Пишет файл через `writeFileAtomic` (уже используется в `characters.js`),
  добавляет запись в `index.json` (`volume: 1.0` по умолчанию), создаёт папку
  `cities/audio/` при первом обращении (`fs.mkdir({recursive:true})`).
- `PUT /api/audio/:id` — тело `{ title? , volume? }` — точечно обновляет поля
  записи в `index.json`. 404, если `id` не найден.
- `DELETE /api/audio/:id` — удаляет файл (не падает, если физически уже
  отсутствует — `fs.unlink().catch(()=>{})`) и запись из `index.json`. 404,
  если `id` не найден.
- Этот роутер использует **собственный** `express.json({ limit: '30mb' })`
  только на `POST /api/audio` (base64 инфлирует 20МБ аудио до ~27МБ) — общий
  лимит сервера (`server.js:71`, сейчас `20mb`) не меняется.
- Раздача файлов: `express.static(path.join(ROOT,'cities','audio'))`
  смонтированный на `/audio-files` — даёт браузеру нативную поддержку
  `Range`-запросов (перемотка в `<audio>`) без ручной реализации.

## Фронтенд

- **`web/public/index.html`**:
  - новый пункт навигации `<a class="nav-item" data-page="audio-library" ...>`
    рядом с существующими (`library`, `tools` и т.д.);
  - новая секция `<section id="page-audio-library" class="page">`:
    заголовок «Звуковая библиотека», кнопка «⏹ Остановить всё», кнопка
    «+ Загрузить звук» (открывает модалку), сетка карточек `#audio-library-grid`.
  - модалка загрузки: поле «Название» (текст, обязательное) + `<input type="file"
    accept="audio/mpeg,audio/ogg,audio/wav">` + кнопка «Загрузить».
- **`web/public/scripts.js`**:
  - загрузка списка при первом открытии страницы (`GET /api/audio`), рендер
    карточек;
  - карточка: название, `<audio loop preload="none">` с `src` на `url` из API,
    кнопка play/pause (обычный toggle — пауза сохраняет позицию, а не
    сбрасывает), слайдер громкости (меняет `audio.volume` вживую + debounce
    `PUT /api/audio/:id {volume}`), кнопка «✎» переименования (инлайн-редактирование
    заголовка карточки), кнопка «🗑» удаления с подтверждением
    (`confirm()`-паттерн, уже используется в проекте для похожих действий);
  - «Остановить всё»: проходит по всем `<audio>` элементам на странице,
    `pause()` + `currentTime = 0`, сбрасывает иконки play/pause в состояние
    «играть»;
  - модалка загрузки: `FileReader.readAsDataURL(file)` → режет `data:audio/...;base64,`
    префикс → `POST /api/audio` с `{ title, filename: file.name, mimetype: file.type, data }`;
    клиентская пре-проверка размера (≤20МБ) и mime до отправки, чтобы не гонять
    большой base64 впустую при заведомо невалидном файле.
- **`web/public/styles.css`**: карточки по образцу существующих `.lib-panel`
  карточек библиотеки (`#page-library`), все размеры через `var(--fs-*)`,
  цвета — только через `:root`-переменные, кнопки карточки ≥44px по высоте
  (touch target).

## Ошибки и крайние случаи

- Загрузка неподдерживаемого формата/файла больше 20МБ — 400 с понятным
  сообщением, показывается тостом в UI (существующий паттерн уведомлений).
- Удаление записи, чей файл на диске уже отсутствует (ручное вмешательство
  извне) — не 500, а тихая очистка записи из манифеста.
- Первое открытие страницы при отсутствии `cities/audio/` вообще — пустой
  список, без ошибки; пустое состояние в UI («Библиотека пуста — загрузите
  первый звук»).

## Тестирование

- Юнит-тест роутера (`web/tests/`) — `GET/POST/PUT/DELETE /api/audio` на
  **временной тестовой директории**, не на реальном `cities/audio/`
  пользователя; синтетический маленький WAV-буфер как фикстура, удаляемый
  после теста.
- Ручная проверка в браузере через существующий скилл `run-sanguine-web`:
  загрузить звук → карточка появилась → play/pause → громкость → переименовать
  → удалить → «Остановить всё» останавливает несколько одновременно играющих
  треков.

## Вне объёма (сознательно не делаем)

- Порядок/сортировка/категории звуков — просто список карточек, без папок/тегов.
- Кросс-браузерная нормализация громкости/эффекты (fade in/out, эквалайзер).
- Общий доступ к библиотеке между несколькими установками/синхронизация в
  облако — библиотека полностью локальна для инсталляции пользователя.
