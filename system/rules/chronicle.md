# ⚙️ СИСТЕМНОЕ ПРАВИЛО: Ведение хроники (VtM v20)

> 📌 Авто-протокол логирования игровых сессий, отслеживания последствий и генерации нарративов. Город-нейтрально: `<город>` — slug города (напр. `paris`).
> ✅ Сверено со структурой `cities/<город>/`.
> ⚠️ Работает в связке с [`diary_rules.md`](diary_rules.md) и [`module_rules.md`](module_rules.md). Ссылки на персонажей/локации/фракции — по карточкам в `cities/<город>/`. Городская специфика — `cities/<город>/rules/`.

---

## 🔁 Протокол внесения записей

При получении отчёта о сессии, черновика событий или запроса на обновление хроники выполни шаги **в строгом порядке**:

1. **Проверка хронологии:** Убедись, что дата/период не пересекается с существующими записями. Если есть конфликт → выдели `⚠️ Хронологический конфликт` и запроси уточнение.
2. **Валидация связей:** Проверь всех упомянутых НПС, локации и фракции по `cities/<город>/archive/characters_index.md`. Если персонаж/место отсутствуют → создай карточку в `cities/<город>/characters/<линейка>/<slug>/` по протоколу из [`npcs_city.md`](npcs_city.md).
3. **Заполнение шаблона:** Внеси данные строго по структуре записей в `cities/<город>/chronicles/<хроника>/events.md` (формат `### 📅 …`). Сохраняй Markdown-разметку, эмодзи-маркеры и отступы. Если для события создан модуль → добавь в конец записи хроники:
   `` > 🔗 [Модуль](../modules/<slug>/<slug>.md) | [Литературный финал](../modules/<slug>/finale.md) ``
   Если финал не создавался — указывать только ссылку на модуль.
4. **Обновление `🌍 Состояние мира`:** Если сессия изменила политический баланс, Маскарад, домены, статус НПС или отношения сект → обнови раздел **`🌍 Состояние мира`** в `cities/<город>/archive/events.md`. Обновить метку «Последнее обновление: [месяц год]».
5. **Внесение в хронику событий:** Добавь запись строго по структуре выше в `cities/<город>/chronicles/<хроника>/events.md` — в конец хронологического блока. Городская сводка `cities/<город>/archive/events.md` пересобирается утилитой `tools/build_city_events.js`.
6. **Создание дневников:** Для каждого участника события (НПС из шага 2) создай или обнови запись в `cities/<город>/characters/<линейка>/<slug>/journal/ГГГГ-ММ.md` строго по правилам из [`diary_rules.md`](diary_rules.md).
   > ⚠️ **Исключения:** персонаж в торпоре или физически отсутствовал → дневник не создаётся или помечается `📝 Косвенная реакция` (подробнее — `diary_rules.md`). Не создавать альтернативных папок и файлов — только эта структура.
7. **Подтверждение:** Верни краткий отчёт:

✅ Хроника обновлена: [Дата/Период]
📍 Локация: [название]
🔗 Затронуты НПС/фракции: [список]
🌍 Изменения состояния мира: [кратко]
📝 Литературная генерация: ✅ Готово
   — или —
📝 Литературная генерация: ⚠️ Ожидает [что именно: отсутствующий НПС / хронологический конфликт / не созданы дневники]

```markdown
<!-- Если параллельная хроника — добавить строку ниже: -->

- **👥 Участники:**
  - [Имя] ([Клан/Род, поколение]) — [роль в сцене]
  - ⚠️ Каждый участник — новая строка; указывать статус «Персонаж Игрока» если применимо
- **📋 События:**
  ⚠️ Один абзац — или разбить на сцены:
  *Сцена 1 — [название]:* [описание].
  *Сцена 2 — [название]:* [описание].
- **⚖️ Последствия:**
  - [Имя / Фракция]: [что изменилось]
  - ⚠️ Открытые вопросы помечать: «открытая нить» или добавить в `cities/<город>/chronicles/<хроника>/open_threads.md`
- **🌍 Изменения состояния мира:**
  - [Персонаж А] ↔ [Персонаж Б]: [новый характер отношений]
  - ⚠️ Если изменилась иерархия, угрозы или домены → также обновить раздел `🌍 Состояние мира` в `cities/<город>/archive/events.md`

> 🔗 [Модуль](modules/<slug>/<slug>.md)
<!-- Если есть литературный финал: -->
> 🔗 [Модуль](modules/<slug>/<slug>.md) | [Литературный финал](modules/<slug>/finale.md)
```

## 📁 Структура хроники (`cities/<город>/chronicles/<slug>/`)

Каждая хроника — отдельная папка со следующими файлами:

| Файл | Назначение |
|---|---|
| `chronicle.md` | Спина хроники: статус, настроение, ссылки на модули |
| `events.md` | Все игровые события хроники (формат `### 📅 …`) |
| `open_threads.md` | Открытые нити (таблица `\| № \| Нить \| … \|`) |
| `modules/<slug>/` | Папки модулей (сессий) |

### Шаблон `chronicle.md`

```markdown
# 📕 Название хроники

- **Статус:** 🟡 Активна        ← 🟡 Активна | 🟢 Закрыта | 🟠 Приостановлена
- **Настроение:** Краткое описание атмосферы и тона хроники

> Спина хроники. События — [events.md](events.md). Нити — [open_threads.md](open_threads.md).
> Закрыть хронику: `node tools/close_chronicle.js <город> <slug> "финал"`

## 🔗 Модули

- [Название модуля](modules/slug_modulya/slug_modulya.md)
- [Следующий модуль](modules/slug2/slug2.md)
```

> ⚠️ Секция `## 🔗 Модули` обновляется **автоматически** при создании/удалении модуля через веб-интерфейс. Не редактировать вручную.

## 📋 Где хранятся события

> ⚠️ События живут в `cities/<город>/chronicles/<хроника>/events.md` (per-event детали) и агрегируются в `cities/<город>/archive/events.md`. Этот файл хранит только протокол (шаги 1–7 выше).
