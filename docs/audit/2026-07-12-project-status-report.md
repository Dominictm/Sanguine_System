## Новое с 07-09 (найдено в этом заходе)

### P1 — утечка внутреннего текста ошибки в `tools.js:740`

```js
} catch (e) {
  res.status(500).json({ ok: false, errors: [e.message] });
}
```

Тот же класс бага, что чинили 07-09 в 11 местах (`serverError()` существует
и **уже используется в этом же файле** 5 раз — строки 609, 649, 680, 694, 800,
844), но эта конкретная точка (эндпоинт применения плана правок,
`POST /api/tools/...apply`, судя по контексту) была пропущена тогда — не
трогалась с 07-09 (`git log` подтверждает отсутствие изменений). `e.message`
здесь может содержать путь к файлу или фрагмент внутренней ошибки записи —
утекает клиенту напрямую вместо общего сообщения с логированием на сервере.

**Рекомендация:** заменить на `serverError(res, e)`, как и остальные 5 мест
в этом файле — однострочная правка, тот же паттерн.

## Рекомендуемый порядок

1. ~~**`tools.js:740` → безопасный фикс**~~ — сделано 2026-07-12 (коммит

   перед правкой проверил фронтенд-обработчик (`lsRunWrite` в scripts.js) —
