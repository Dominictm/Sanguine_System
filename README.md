# Sanguine System

Структурированная система ведения хроник **World of Darkness** (Vampire: The Masquerade V20 и смежные линейки) с **ИИ-Рассказчиком** и локальным веб-интерфейсом для просмотра, генерации и ведения игры.

Нейтральный шаблон: **никакого преднастроенного контента**. Скачай, запусти, создай свой город и начинай хронику — несколько городов одновременно поддерживаются из коробки.

ИИ-функции работают через **любого** из провайдеров: Claude Code (подписка), Anthropic API, OpenRouter (бесплатные модели) или OpenAI (GPT) — и переключаются по каждой функции отдельно.

> 📘 Пошаговое руководство по работе с проектом — **[GUIDE.md](GUIDE.md)**.

---

## Требования

| Компонент | Версия | Назначение |
|---|---|---|
| [Node.js](https://nodejs.org) | 18+ | Веб-интерфейс, утилиты (`tools/*.js`), создание города |
| PowerShell | 5.1+ (Windows) | Часть утилит (`validate_links`, `search`) — опционально |
| **ИИ-провайдер** (любой) | — | Нужен только для генерации; просмотр/правка работают и без него |

**Провайдеры ИИ** (достаточно одного):

| Провайдер | Авторизация | Стоимость |
|---|---|---|
| [Claude Code](https://claude.com/claude-code) | вход по подписке (OAuth) или `ANTHROPIC_API_KEY` | по подписке / API |
| [OpenRouter](https://openrouter.ai) | `OPENROUTER_API_KEY` | есть **бесплатные** модели |
| [OpenAI (GPT)](https://platform.openai.com) | `OPENAI_API_KEY` | по API |

Всё настраивается в вебе: **Инструменты → 🤖 Модели AI** (или вручную в `web/.env`).

## Быстрый старт

**1. Создай свой первый город:**

```bash
node tools/new_city.js london "Лондон" 2010
```

Создаст каркас `cities/london/` (сеттинг, архив, линейки персонажей, хроники, локации).
Затем опиши сеттинг в `cities/london/city.md` (политика, фракции, локации, лейтмотивы).

**2. Запусти веб-интерфейс:**

```bash
cd web && npm install && node server.js      # → http://localhost:3000
```
(на Windows можно запустить `start.bat` из корня проекта)

В шапке слева выбери свой город в выпадающем списке.

**3. Настрой ИИ** (опционально, для генерации) — **Инструменты → 🤖 Модели AI**:
залогинься через Claude Code либо вставь ключ OpenRouter/OpenAI, затем в **«⚡ Назначение провайдеров»** выбери провайдера и модель для каждой функции.

**4. Добавляй персонажей** — по протоколу `system/rules/npcs_city.md` (через ИИ или вручную),
карточки кладутся в `cities/<город>/characters/<линейка>/<slug>/<slug>.md`.

**5. Веди модули и сессии** — создай модуль, сгенерируй его, логируй сессии по ходу игры и закрой по завершении (фазы A/B/C, см. ниже).

## ИИ-провайдеры и авторизация

Каждая ИИ-функция (см. таблицу ниже) индивидуально привязывается к провайдеру и модели в разделе **Инструменты → 🤖 Модели AI → ⚡ Назначение провайдеров**. Это позволяет, например, писать прозу сильной моделью Claude, а короткие реплики НПС — бесплатной моделью OpenRouter.

**Авторизация провайдеров** (в той же вкладке):

- **🌐 OpenRouter** — поле `OPENROUTER_API_KEY`. В списках моделей доступны **все бесплатные** модели live-каталога, плюс автоматический `openrouter/free`.
- **🤖 OpenAI (GPT)** — поле `OPENAI_API_KEY` (у OpenAI доступ к API только по ключу; входа «через аккаунт ChatGPT» для API нет).
- **🧠 Claude (Anthropic)** — два варианта:
  - **Вход через Claude Code** (без API-ключа): кнопка открывает страницу авторизации Claude, после входа вставляешь показанный код — токен подписки сохраняется локально (`~/.claude/.credentials.json`). Истёкший токен обновляется автоматически.
  - **API-ключ** `ANTHROPIC_API_KEY` — как альтернатива.
  - Кнопка **«Обновить статус»** перечитывает состояние входа (например, после `/login` в Claude Code в терминале).

**Ключи в `web/.env`** (всё, что задаётся в UI, пишется сюда):

```
OPENROUTER_API_KEY=sk-or-...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...     # опционально, вместо входа через Claude Code
```

**Отказоустойчивость:** при лимите запросов (429) генерация делает повторы с задержкой и автоматически переключается на резервного провайдера (Claude → GPT → OpenRouter), чтобы не падать ошибкой; в ответе помечается, если результат пришёл от резерва.

## ИИ-функции

| Функция | Где | Что делает |
|---|---|---|
| **Проза дневников и финалов** | Сессия · Закрытие модуля | Литературные записи по `diary_rules.md` (клановый стиль, Маскарад, факты из событий) |
| **Генерация модуля** | Страница модуля | По концепции строит сценарий, локации и эпизодических НПС; проверяет таймлайн (даты/статусы), вплетает связи персонажей, переиспользует существующие НПС/локации |
| **Реплики НПС** | Карточка персонажа · вкладка «НПС» модуля | Диалог в характере по полю **Голос** + клановому стилю для заданной ситуации |
| **Проверка непротиворечий канону** | Панель целостности · форма сессии | Сверяет текст события с фактами (статусы, даты, «мёртвые» в сцене) и подсвечивает противоречия |
| **«Ранее в хронике…»** | для игроков | ИИ-рекап последних событий хроники |
| **Внешность по арту** (Vision) | Карточка персонажа | Описание внешности по загруженным изображениям |
| **Промт для изображения** | Карточка персонажа | Генерация positive/negative-промта по секции «Внешность» |

> Провайдер и модель для каждой функции выбираются независимо в «⚡ Назначение провайдеров».

## Модули и сессии

Жизненный цикл модуля — три фазы (правила: `system/rules/module_rules.md`):

- **Фаза A — создание и генерация.** Кнопка «+ Модуль» создаёт стаб с разделом `## 💡 Концепция`. Кнопка «🪄 Сгенерировать» по концепции строит `scenario.md`, карточки локаций и эпизодических (модульных) НПС в `npc/<slug>/`.
- **Фаза B — ведение во время игры.** Вкладка **«🎲 Сессии»**: журнал по сессиям (не каждый модуль проходится за один вечер). Сцены выбираются из сценария (сыгранные исключаются), плюс постоянный пункт «События вне сценария»; записи можно редактировать.
- **Фаза C — закрытие.** Кнопка «🔒 Закрыть модуль» генерирует литературный финал и переносит каноничное событие в `events.md` хроники. Кнопка «Удалить модуль» вычищает записи модуля из хроник, ссылки в дневниках и неканоничных НПС, созданных только в нём.

**Вкладки страницы модуля:** Краткое · Сценарий · Сессии · События · НПС · Локации · Нити.
На вкладке **НПС** имя каждого персонажа — ссылка на карточку, а под НПС есть окно «💬 Реплика НПС» для генерации ответа в характере.

## Структура

```
.
├── CLAUDE.md                     # Протокол Рассказчика (город-независимый) + указатель активного города
├── cities/                       # ★ Города (создаются через tools/new_city.js)
│   └── <город>/
│       ├── city.md               # Сеттинг города
│       ├── rules/                # Правила, специфичные городу
│       ├── archive/              # political_state · events (агрегат) · timeline · characters_index · visitors
│       ├── characters/<линейка>/<slug>/   # <slug>.md, <slug>-sheet.md, art/, journal/{ГГГГ-ММ.md, retrospective.md}
│       ├── locations/district_NN/<район>/<локация>/
│       └── chronicles/<хроника>/          # chronicle · events.md · open_threads.md
│           └── modules/<модуль>/          # <модуль>.md · scenario.md · sessions.md · finale.md · npc/<slug>/
│
├── system/                       # Глобальное (одно на все города)
│   ├── rules/                    # npcs_city · module_rules · diary_rules · chronicle · character_sheet_v20 · portret · CHECKLIST
│   └── schema/                   # card_schema.md (контракт карточки) · validate_cards.js (линтер)
│
├── tools/                        # Утилиты
│   ├── new_city.js               # ★ Создать новый город
│   ├── build_city_events.js      # Пересобрать индекс событий города
│   ├── validate_links.ps1        # Проверка внутренних ссылок (Windows)
│   └── search.ps1 / update_images.ps1 / …
│
└── web/                          # Веб-интерфейс (Node + Express)
    ├── server.js                 # API (city-aware, ?city=); ИИ-маршрутизация (OpenRouter/OpenAI/Claude)
    ├── lib/parsers.js            # Парсеры карточек/локаций/связей
    ├── .env                      # Ключи провайдеров (не в git)
    └── public/                   # SPA: панель, хроника, персонажи, граф, модули, локации, сессия, настройки ИИ
```

### Карточка персонажа

Контракт полей — `system/schema/card_schema.md`. Проверка — `node system/schema/validate_cards.js`.
Обязательно: H1 (эмодзи + имя), `Слаг`, `Родной город`, `Линейка WoD`, `Статус`, секция `## 🖼️ Изображения`.
Имя — кириллицей в карточке; папка — ASCII-слаг.

## Веб-интерфейс

| Раздел | Описание |
|---|---|
| **Панель** | Статистика выбранного города + панель целостности (битые ссылки, проверка канона) |
| **Хронология** | Состояние мира + лента событий (агрегат хроник) |
| **Персонажи** | Карточки с фильтрами; двойной клик — деталь (внешность, лист, дневники, реплики НПС) |
| **Связи** | D3.js-граф отношений; типы связей, редактирование по записям |
| **Фракции** | Карта политических сил города |
| **Хроники** | Список хроник города |
| **Модули** | Модули по хроникам → страница модуля (фазы A/B/C) |
| **Нити** | Открытые сюжетные нити |
| **Слухи** | Слухи Элизиума / Грёзы (d20) |
| **Локации** | Карточки локаций |
| **Инструменты** | Новый домен · Новый НПС · Модуль · 📓 Сессия · 🛠 Ещё · Проверка · 🤖 **Модели AI** |
| **Поиск** | Полнотекстовый поиск по городу |

Селектор города слева переключает весь интерфейс (`?city=<slug>`).

---

## Мультигород

Каждый город самодостаточен (`cities/<город>/`). Создавай сколько нужно через `new_city.js`,
переключай в вебе. Персонаж принадлежит одному `Родной город`; присутствие в другом городе
оформляется ссылкой в `cities/<другой>/archive/visitors.md` (один источник истины, без копий).
