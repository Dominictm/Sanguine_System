# Спека — «Добавить в хронологию мира» из финала модуля/хроники

Дата: 2026-07-13
Фаза 1 дорожной карты `docs/superpowers/plans/2026-07-13-development-roadmap.md`.

## Задача

Итог закрытого модуля (или всей хроники) попадает в `cities/<city>/archive/timeline.md`
одним подтверждением — через предзаполненную форму строки эпохи, без ручного
перезаполнения на вкладке «Хронология мира».

## Решения (подтверждены пользователем)

- Кнопка — **в обеих точках показа финала модуля** + **у хроники**.
- Новых серверных эндпойнтов **нет** — запись через существующий
  `POST /api/timeline/epoch/:heading/row` (`web/routes/archive.js`).

## Точки входа (3)

1. **Модалка «📜 Финал» модуля** — `openFinalePreview()`
   (`web/public/scripts/modules.js:933`, модалка `#finale-preview-modal`):
   кнопка «→ В хронологию мира» в футере модалки, рядом с «Закрыть».
2. **Блок «Финал модуля» в раскрытом событии** — `_loadEventFinale()`
   (`web/public/scripts/archive.js:1125`): кнопка после текста финала
   внутри `.chron-finale`.
3. **Шапка модалки деталей хроники** (`#chr-detail-*`, рендер
   `openChronicleDetail` в `modules.js`): кнопка в шапке рядом с вкладками.
   Текст финала хроники в UI сейчас не отображается нигде — его рендер
   **вне рамок** этой фазы; кнопка доступна независимо от статуса хроники
   (Рассказчик сам решает, когда итог готов).

Вне рамок: вкладка «Финал» в `openModuleDetail` (`scripts.js:1578`) —
третья точка показа финала модуля; не добавляем, чтобы не размазывать
кнопку по всем представлениям (обе согласованные точки покрывают сценарий
«прочитал финал → занёс в хронологию»).

## Общая модалка формы

Новая модалка `#timeline-add-modal` в `web/public/index.html`; логика — в
`web/public/scripts/archive.js` (рядом с helpers форм хронологии, глобальная
функция `openTimelineAddModal(prefill)`), т.к. там уже живут
`_timelineData`, построители форм и пикер связей.

`prefill = { title, linkText, linkHref }`:
- модуль: `title` = заголовок модуля, `linkText` = название модуля,
  `linkHref` = `../chronicles/<chr>/modules/<mod>/<mod>.md`;
- хроника: `title` = отображаемое имя хроники,
  `linkHref` = `../chronicles/<slug>/chronicle.md`.

(Пути — относительно `archive/timeline.md`, тот же формат, что у
`_resolveTimelineLink` для персонажей/локаций.)

### Поля

| Поле | Контрол | Предзаполнение |
|------|---------|----------------|
| Эпоха | select из `epochs[].heading` (`GET /api/timeline/structured`) | последняя эпоха |
| Год | text (`form-control`) | год города из `GET /api/cities` (кэш `STATE`); пусто, если не задан |
| Тип | select из `legend[]` того же ответа (символ + значение) | 🏛️, если есть в легенде, иначе первый |
| Событие | textarea | `**<title>.** ` (курсор в конец) |
| Источник | select 📚 / 🏙️ / ❓ | 🏙️ |
| Связи | предзаполненный чип-линк на модуль/хронику + существующий пикер персонажей/локаций (`_renderLinkChipsInto`, `_pendingLinks`) | ссылка на источник |

Поля переиспользуют классы/разметку существующей формы строки
(`_timelineRowFormHtml`) — не параллельная копия стилей, а тот же
`form-control` + `tl-form-*`.

### Поведение

- Сохранение: `POST /api/timeline/epoch/<heading>/row` с телом строки в
  формате существующего эндпойнта. Успех → тост + закрытие модалки.
- Ошибки: пустой год/событие → инлайн-подсказка, запрос не уходит
  (та же валидация, что в форме вкладки); 404/409/5xx от API → сообщение
  в модалке, введённые данные не теряются.
- Нет ни одной эпохи в `timeline.md` → вместо формы сообщение
  «Сначала создайте эпоху на вкладке Хронология мира» и кнопка-ссылка
  на вкладку.
- Повторное добавление той же строки не блокируется (дедупликация —
  ответственность Рассказчика).
- Кнопки — существующий стиль (`chron-toggle`-семейство/кнопки модалок),
  никаких новых цветов/размеров вне `:root`-токенов.

## Что НЕ делается

- Никаких изменений сервера (`web/routes/**`, `web/lib/**`).
- Никакого автоизвлечения года/выжимки из текста финала AI-ем.
- Рендер финала хроники в UI.

## Тесты / приёмка

Серверной логики нет → новых unit/integration-тестов нет. Приёмка — CDP-сценарий
(по `.claude/skills/run-sanguine-web`):

1. Открыть «📜 Финал» существующего модуля → кнопка видна → форма
   предзаполнена (эпоха/год/тип/событие/чип-ссылка).
2. Сохранить строку с маркером `__CDP_TEST__` → строка появляется в
   `timeline.md` нужной эпохи (проверка через `GET /api/timeline/structured`).
3. То же из раскрытого события и из шапки хроники (кнопки открывают ту же
   модалку с корректным prefill).
4. Ошибок в консоли нет; тестовые строки откачены (`git diff` по `cities/` пуст).
5. Полный прогон `npm run test:unit` зелёный (регрессий нет),
   `web/tests/report.html` откачен.
