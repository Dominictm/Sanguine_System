# Card Schema — карточка персонажа

> Контракт обязательных полей карточки. Проверяется `validate_cards.js`.
> Базовый формат — буллеты `- **Поле:** значение` (не frontmatter), как в `system/rules/npcs_city.md`.

---

## Расположение

| | Текущее | Целевое (после реструктуризации) |
|---|---|---|
| Главная карточка | `characters/<lineage>/<Имя>/<Имя>.md` | `cities/<city>/characters/<lineage>/<slug>/<slug>.md` |
| Лист V20 | `<Имя>/<Имя>-лист.md` | `<slug>/<slug>-sheet.md` |
| Дневники | `<Имя>/Journal_<Имя>/ГГГГ-ММ.md` | `<slug>/journal/{retrospective.md, ГГГГ-ММ.md}` |
| Изображения | `<Имя>/*.png` (рядом) | `<slug>/art/*.png` |

Главная карточка определяется по правилу: имя файла = имя папки (`<slug>/<slug>.md`).
Файлы `-sheet`, папки `journal/`, `art/` — не карточки.

---

## Обязательные элементы

| Элемент | Правило | Пример |
|---|---|---|
| **H1** | `# <эмодзи> <display_name>` | `# 🎪 Верене де Кюстин` |
| **Нав-ссылка** | строка `> 🔗 [Все персонажи](...)` | (+ опц. ссылка на лист) |
| **`Линейка WoD`** | = имя под-папки линейки (см. мэппинг) | `Вампир` → `vampires` |
| **`Статус`** | значение ∈ enum статусов | `Жива` |
| **`## 🖼️ Изображения`** | секция в конце; список арта или `⏳ Изображение не предоставлено` | |

### Новые обязательные поля (вводятся реструктуризацией)

| Поле | Назначение | Пример |
|---|---|---|
| **`Слаг`** | ASCII-ключ папки (`[a-z0-9_]`) | `veren_de_kustin` |
| **`Родной город`** | город-владелец (display) | `Париж` |
| **`Пол`** | ∈ enum `Мужской`, `Женский`, `Неизвестно`; обязателен при создании персонажа (CLI/веб). `Неизвестно` — только когда неопределимость пола — часть лора персонажа (см. инвариант 8), не «забыли заполнить» | `Женский` |

### Поле Принадлежность

| Значение | Описание |
|---|---|
| `Персонаж мастера` | Персонаж создан Мастером как НПС (по умолчанию) |
| `Персонаж игрока` | Постоянный игровой персонаж (ПК) |
| `Эпизодический персонаж` | Создан автоматически при генерации модуля; может быть канонизирован |
| `Фамильяр` | Компаньон, привязанный к хозяину: животное-гуль вампира, фамильяр-дух мага (он же у Тремера-экс-мага) либо химерический спутник подменыша. Заводится отдельной полноценной карточкой; линейка выбирается по природе существа — см. «Справка по правилам» в плане |

Поле используется для фильтрации при создании модулей, генерации дневников и управлении персонажами.
Добавляется в шаблон как: `- **Принадлежность:** Персонаж мастера`

У персонажа с Принадлежностью `Фамильяр` хозяин (домитор-вампир, маг, подменыш) указывается
**первым буллетом в секции `Отношения`** со ссылкой на карточку и явным словом «домитор» /
«хозяин». Отдельного поля «Хозяин» нет намеренно: связь и так даёт ребро в графе через
`Отношения`, а механическая сторона живёт в Дополнении «Домитор»
(`system/library/backgrounds/ghoul.json`), где шкала 1-5 задаёт поколение домитора и потолок
доступных фамильяру Дисциплин.

### Новые опциональные поля

| Поле | Когда | Пример |
|---|---|---|
| **`Алиасы`** | у персонажа есть вторая личина/кличка | `Кален Урес` |
| **`Присутствие`** | персонаж появлялся вне родного города | `Лос-Анджелес — summer_2011 — 2011-06` |
| **`Хочет`** | мотивация НПС (проактивность в генерации) | `Место в Совете Примогенов` |
| **`Боится`** | страх НПС — материал для конфликтов | `Раскрытие старого предательства` |
| **`Рычаг`** | чем на НПС можно надавить | `Долг перед Шерифом` |

Тройка `Хочет/Боится/Рычаг` (E2, 2026-07-15): при заполнении генерация
(сценарии, реплики) считает НПС проактивным — он делает шаги к цели в
сценах без игроков. По одной короткой фразе на поле.

`display_name` отдельным полем не хранится — это текст H1 без ведущего эмодзи.
Фон/референс изображения (бывш. `npc_image_mapping.md`) переносится в секцию `## 🖼️ Изображения` строкой `<!-- фон: <цвет/описание> -->` или подписью к арту.

## Промт изображения — два канонических формата

Блок промта читается/пишется веб-интерфейсом через единые `readPrompt` / `writePrompt` (`web/server.js`). Поддерживаются **два** формата — выбор зависит от типа карточки. Не смешивать их в одной карточке.

**Формат A — «indented» (карточки персонажей).** Буллет-метка + строки-продолжения с отступом в 2 пробела:

```markdown
- **🎨 Промт для генерации изображения:**
  [Блок 1] …
  [Блок 2] …
  [Блок 3] …
- **🚫 Негативный промт:**
  …
```

**Формат B — «fenced» (карточки локаций).** Жирная метка + блок ` ``` `:

````markdown
## 🎨 Промт для генерации изображения

**GPT / DALL-E 3:**
```
[позитивный промт]
```

**Негативный промт (SD / Flux):**
```
[негативный промт]
```
````

Подробности по содержимому блоков — `system/rules/portret.md` (персонажи — три блока; локации — три блока + фиксированный хвост).
`readPrompt` распознаёт оба формата; `writePrompt` сохраняет карточку в её текущем формате (или дописывает блок в указанном формате, если его не было).

## Инварианты (проверяет линтер)

1. У каждой папки персонажа есть главная карточка `<slug>/<slug>.md`.
2. Присутствуют H1, нав-ссылка, `Линейка WoD`, `Статус`, секция `## 🖼️ Изображения`.
3. `Статус` ∈ enum.
4. `Линейка WoD` соответствует под-папке линейки.
5. **`Слаг`** уникален в пределах города; **`Алиасы`** не пересекаются с чужими `display_name`/`Слаг`/`Алиасы` в городе.
6. `Родной город` присутствует и указывает на существующий город.
7. `Пол` ∈ enum (`Мужской`, `Женский`, `Неизвестно`).
8. `Пол: Неизвестно` допустим только при явном нарративном обосновании в карточке (намеренная неопределимость как часть личины/природы персонажа), не как заглушка для незаполненного поля.

> До Фазы 5 поля `Слаг`/`Родной город` отсутствуют в текущих карточках — линтер по умолчанию выдаёт по ним **WARNING**, а с флагом `--strict` — ERROR (включается после Фазы 5).
> `Пол` — новое поле (введено вместе с обязательностью при создании персонажа): в карточках, созданных до этого момента, отсутствует — линтер по нему выдаёт только **WARNING** (без `--strict`-апгрейда, до отдельного решения о бэкфилле).
