# Спека — экран сессии (режим живой игры)

Дата: 2026-07-13
Фаза 4 дорожной карты `docs/superpowers/plans/2026-07-13-development-roadmap.md`.

## Задача

Одна страница для ведения живой сессии — без прыжков по вкладкам: сценарий
активного модуля, быстрый доступ к НПС, запуск аудио-пресета, заметки,
передача итогов в запись сессии. Дайс-роллер (фаза 3) уже глобальный (fab).

## Состав (3 независимо сдаваемых куска)

### A. Каркас: страница + выбор модуля + сценарий по сценам

- Пункт навигации «Сессия» (🎬, `data-page="session"`) после «Модули»;
  секция `#page-session`; роутинг в `navigate()` (`scripts.js`) →
  `loadSessionScreen()` в новом `web/public/scripts/session-screen.js`.
- Выбор: select хроники (`GET /api/chronicles`, только активные первыми) →
  select модуля (`GET /api/chronicles/:slug/modules`). Выбор запоминается в
  `localStorage` (`sanguine.session.<city>`) и восстанавливается.
- Сценарий: `GET .../modules/:mod/detail` → поле `scenario` (raw md),
  режется на блоки по `## `-заголовкам (тот же приём, что модульная
  страница); блоки — аккордеон, один развёрнут; кнопки «← Сцена / Сцена →»;
  текущая сцена запоминается в том же localStorage-ключе.
- Новых серверных эндпойнтов нет.

### B. НПС + аудио

- Колонка «НПС модуля»: из `/detail` (`npcs`, `pcs`) — чипы; клик по чипу
  с карточкой в городе открывает существующую модалку персонажа
  (`openCharDetail`); без карточки — просто текст.
- Блок «Аудио»: `GET /api/audio/presets` — пресеты, чьи `locationSlug`
  входят в локации модуля (`/detail.locations`), плюс остальные ниже;
  кнопка ▶ пресета: если аудио-библиотека ещё не загружалась —
  `await loadAudioLibrary()` (карточки-`<audio>` живут в скрытой секции
  `#page-audio-library`, играть можно с любой страницы), затем
  `_audioPresetPlay(id)`; кнопка ⏹ — `_audioPresetStop()`.

### C. Заметки + передача в запись сессии

- Textarea «Заметки сессии», автосохранение в тот же localStorage-ключ
  (debounce), переживает перезагрузку.
- Кнопка «→ Записать сессию»: переход на Инструменты → вкладка
  «📓 Сессия» (существующая форма `ls-*`) с предзаполнением `#ls-summary`
  заметками (только если поле пусто — не затирать ручной ввод) и
  хроники/модуля, если поля пусты. Пайплайн записи не дублируется.

## Ограничения

- Никаких новых серверных эндпойнтов; только композиция существующих API.
- CSS — токены; touch ≥44px; двухколоночная сетка сворачивается в одну
  колонку на узких экранах (existing breakpoints конвенция).
- `_v20Model`-подстановка пула в роллере с этого экрана не работает (лист
  не открыт) — это ок, роллер остаётся ручным.

## Тесты / приёмка

Серверных изменений нет → новых unit-тестов нет. CDP: выбор
хроники/модуля → сценарий разбит на сцены, навигация сцен работает;
чип НПС открывает модалку; пресет запускается (audio.paused === false) и
останавливается; заметки переживают перезагрузку; «→ Записать сессию»
открывает форму с предзаполнением; консоль чистая; `npm run test:unit`
зелёный (регрессий нет).
