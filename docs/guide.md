## 1. Запуск и первый старт

**Двойной клик на `start.bat`** — единственное, что нужно сделать.

Скрипт автоматически:
- Проверяет наличие Node.js и Git, при необходимости устанавливает
- Запускает TgWsProxy (если есть в корне — нужен для Gemini AI)
- Ставит npm-зависимости при первом запуске
- Открывает браузер на `http://localhost:3000`

Окно терминала держи открытым — закрытие останавливает сервер.
Для перезапуска сервера без закрытия окна: **Инструменты → Модели AI → Перезапустить сервер**.

### Первоначальная настройка

1. **Создай город** — если городов нет, в левой панели появится подсказка.
   Перейди в **Инструменты → Домены** и заполни форму создания.

2. **Настрой ИИ** (опционально) — **Инструменты → Модели AI**.
   Без ИИ-провайдера интерфейс работает полностью, только генерация недоступна.

3. **Добавь персонажей** — через **Инструменты → Новый НПС** или вручную (папка `cities/<город>/characters/`).

## 3. Города (домены)

Каждый город — самостоятельная единица. Можно вести несколько городов одновременно.

### Создание города

**Инструменты → Домены → Создать новый домен**

Поля формы:
- **Название** — русскоязычное имя (например, «Париж»)
- **Год** — игровой год сеттинга (например, 2010)
- **Описание сеттинга** — краткое описание атмосферы и особенностей
- **Политика** — описание фракций и расстановки сил
- **Ключевые локации** — главные места силы
- **Лейтмотивы** — темы и атмосфера хроники
- **Местная специфика** — язык, имена, обычаи
- **Запреты** — что нельзя делать в этом городе (для ИИ)
- **Источники** — канонические материалы
- **Районы** — список районов (каждый получит папку)

После создания город появится в выпадающем списке.

### Персонаж из другого города

Если персонаж гостит в чужом городе: **Инструменты → Ещё → 🌍 Кросс-город**.
Данные пишутся в `cities/<город>/archive/visitors.md` — один источник истины без копий карточки.

## 4. Персонажи

### Просмотр

Раздел **Персонажи** показывает карточки всех персонажей города.

**Фильтры:**
- Поиск по имени (в реальном времени)
- По линейке: Вампиры / Феи / Смертные / Оборотни / Маги / Охотники
- По статусу: Активен / Летаргия / Мёртв / Пропал / Неизвестен

Карточки с несколькими изображениями автоматически листают арт каждые 12–50 секунд.

### Создание персонажа

Кнопка **+ Новый персонаж** (или **Инструменты → Новый НПС**):

1. Выбери линейку WoD
2. Заполни поля: имя (кириллица), клан, фракция, поколение, год рождения / Объятия, Сир
3. Биография, внешность (3–5 визуальных маркеров)
4. Загрузи арт (опционально)

Персонаж создаётся с правильной структурой папок и обновляет `characters_index.md`.

### Карточка персонажа (детальный вид)

Клик по персонажу открывает панель с вкладками:

#### Вкладка «Информация»
Основные данные из карточки: клан, фракция, статус, поколение, связи. Фон подкрашивается цветом клана.

#### Вкладка «Биография»
Полный текст биографии с редактором. Кнопка **✍️ Сгенерировать ИИ** — сгенерировать описание по полям карточки.

#### Вкладка «Отношения»
Список связей с другими персонажами. Типы: Семья · Сир/Дитя · Союзник · Враг · Верность · Романтика · Подозрение · Знакомый · Тайное · Нейтрально.
Добавляются и редактируются прямо в этой вкладке.

#### Вкладка «Дневники»
Список дневниковых записей по периодам (формат `ГГГГ-ММ` или `retrospective`).

- Клик по периоду — открывает запись
- **✍️ Сгенерировать ИИ** — генерирует литературную запись по полям (период, заголовок сессии, подсказка-настроение)
- Запись редактируется перед сохранением
- Удаление — кнопка корзины

#### Вкладка «Лист V20»
Полный игровой лист персонажа: атрибуты, навыки, дисциплины, добродетели, здоровье, воля, кровь, фоны, преимущества/недостатки.

- **📋 Сгенерировать лист** — ИИ заполняет числа по карточке и биографии
- **♻ Перегенерировать ИИ** — перезаписать числа из нового ИИ-запроса
- Все поля редактируются вручную, изменения сохраняются автоматически
- Клик по дисциплине — открывает справочник с описанием уровней

#### Вкладка «Описание»
Вся карточка персонажа в raw-виде с редактором. Полный контроль над markdown.

#### Блок «💬 Реплики НПС»
Генерирует реплики в характере персонажа для заданной ситуации.
Голос, клановый стиль и факты берутся из карточки — никаких выдуманных деталей.

**Кнопки в заголовке:**

| Кнопка | Что делает |
|--------|------------|
| **🎭 Директор** | ИИ анализирует текущее состояние и предлагает варианты следующей сцены |

## 6. Модули и сессии

Модуль = одна история (может занимать несколько игровых вечеров). Жизненный цикл — три фазы.

### Фаза A — Подготовка

1. В хронике нажми **+ Модуль**
2. Заполни название и раздел `## 💡 Концепция` в модуле
3. Открой модуль → кнопка **🪄 Сгенерировать**

**Форма генерации сценария:**
- Список персонажей-игроков (ПК)
- Список НПС, задействованных в модуле
- Текст концепции → ИИ пишет полный `scenario.md`

ИИ учитывает: текущие статусы персонажей, открытые нити, даты в таймлайне — и предупреждает о противоречиях.

### Фаза B — Ведение игры

Страница модуля, вкладка **🎲 Сессии**.

Кнопка **📓 Записать сессию** → форма лога:

**Секции лога:**
- **Модуль** — привязка к модулю (новый или существующий)
- **Хроника** — к какой хронике относится
- **Событие** — месяц `ГГГГ-ММ`, дата текстом, заголовок, локация
- **Сцены** — список сцен с НПС, локацией и ключевыми моментами
- **Последствия** — изменения в мире: смерти НПС, сдвиги фракций, новые факты
- **Участники** — ПК и НПС, кому сгенерировать дневник
- **Нити** — новые нити и закрытые нити (по ID)

После заполнения — кнопка **Предпросмотр**, затем **Записать** — данные пишутся в файлы.

### Фаза C — Закрытие модуля

Кнопка **🔒 Закрыть модуль** на странице модуля.

Действие запрашивает подтверждение, затем:
1. Генерирует литературный финал (`finale.md`)
2. Переносит каноничное событие в `events.md` хроники
3. Закрывает оставшиеся нити модуля

**Кнопка «Удалить модуль»** — вычищает записи модуля, ссылки в дневниках и модульных НПС.

### Вкладки страницы модуля

| Вкладка | Содержимое |
|---------|------------|
| **ℹ Краткое** | Сводка: ПК, НПС, локации, описание |
| **📝 Сценарий** | Полный текст сценария с кнопкой 🪄 Сгенерировать |
| **🎲 Сессии** | Лог сессий + кнопка записи |
| **📅 События** | События, созданные в этом модуле |
| **👥 НПС** | Карточки НПС с кнопкой реплик для каждого |
| **📍 Локации** | Локации, задействованные в модуле |
| **🧵 Нити** | Открытые нити из этого модуля |

## 7. Нити

**Нити** — открытые сюжетные зацепки и незакрытые арки.

### Статусы нитей

| Статус | Значение |
|--------|----------|
| 🔴 Активная | Требует внимания прямо сейчас |
| 🟡 Фоновая | Тлеет в фоне, может вспыхнуть |
| 🟢 Закрытая | Разрешена |
| ⚫ Заброшенная | Дроп без разрешения |

### Создание нити

Кнопка **+ Нить** (или автоматически через лог сессии):
- Заголовок, приоритет (Высокий / Средний / Низкий)
- Описание
- Источник (модуль + период)
- Целевой файл

Нити из лога сессии создаются автоматически при записи.

## 8. Локации

**Локации** — карточки мест: злачные клубы, Элизиумы, логова, нейтральные территории.

### Фильтры

- Поиск по названию
- **Зона**: Элизиум · Носферату · Нейтральная · Опасная · Другое
- **Район**: автозаполнение из структуры папок

### Создание локации

**Инструменты → Ещё → 📍 Новая локация** или кнопка **+ Локация** в разделе.

### Детальный вид локации

| Вкладка | Содержимое |
|---------|------------|
| **Метаданные** | Район, адрес, зона, под чьим контролем |

## 14. ИИ-генерация

### Настройка провайдеров

**Инструменты → Модели AI → ⚡ Назначение провайдеров**

Каждая функция генерации настраивается независимо:

| Функция | Описание |
|---------|----------|
| **Внешность по арту** | Vision-анализ загруженного изображения |
| **Генерация локаций** | Карточки мест (атмосфера, VtM-контекст, крючки) |
| **Генерация прозы** | Дневники и финалы сессий |
| **Биография** | Текст биографии персонажа |
| **Реплики НПС** | Диалог в характере |
| **Лист V20** | Числа атрибутов/навыков по карточке |
| **Сценарий модуля** | Полный сценарий по концепции |

Для каждой функции выбираешь: **провайдер** (OpenRouter / GPT / Claude / Gemini) и **модель**.

### Доступные провайдеры

| Провайдер | Авторизация | Где настроить |
|-----------|-------------|---------------|
| **Claude** | Кнопка «Войти через Claude» или `ANTHROPIC_API_KEY` | Инструменты → Модели AI |
| **Gemini** | `GEMINI_API_KEY` + TgWsProxy | [docs/gemini.md](gemini.md) |
| **OpenRouter** | `OPENROUTER_API_KEY` | Инструменты → Модели AI |
| **GPT** | `OPENAI_API_KEY` | Инструменты → Модели AI |

### Резервный провайдер

При перегрузке (ошибка 429) генерация автоматически переключается по цепочке:
`Claude → GPT → OpenRouter`. В ответе появляется пометка «⚠️ сгенерировано резервным».

### Функции генерации по разделам

#### Проза дневника (карточка персонажа → Дневники)
Кнопка **✍️ Сгенерировать ИИ**. Поля:
- Период `ГГГГ-ММ`
- Заголовок сессии (контекст)
- Подсказка-настроение

Результат — редактируемый черновик перед сохранением.

#### Реплики НПС (карточка персонажа)
Блок **💬 Реплики НПС** → введи ситуацию → несколько реплик в характере, голосе и клановом стиле.

#### Лист V20 (карточка → вкладка «Лист V20»)
**📋 Сгенерировать лист** — ИИ назначает числа атрибутам, навыкам, дисциплинам по карточке и биографии.
**♻ Перегенерировать** — пересчитать числа (существующие перезапишутся).

#### Сценарий модуля (страница модуля)
**🪄 Сгенерировать** → форма с ПК, НПС и концепцией → полный `scenario.md`.
ИИ проверяет таймлайн и статусы персонажей. Предупреждения о противоречиях выводятся отдельно.

#### AI-Директор (хроника)
**🎭 Директор** — анализирует состояние модуля и предлагает 3–5 вариантов следующей сцены с учётом напряжённости и открытых нитей.

#### Recap (хроника)
**📺 Ранее в хронике…** — генерирует «ранее в нашей хронике» для зачитывания игрокам в начале сессии.

#### Проза сессии (лог сессии)
После записи сессии — кнопка **🪄 Сгенерировать прозу** превращает структурированный лог в художественный текст финала.

## 15. Инструменты

Раздел **Инструменты** — семь вкладок:

### Домены
Список городов с детальным видом. Форма создания нового домена (см. раздел 3).

### Новый НПС
Мастер создания персонажа через форму (без ручного создания файлов).

### Модуль
Перенаправляет к созданию модуля через хронику (кнопка «+ Модуль»).

### 📓 Сессия
Форма записи игровой сессии (см. [Фаза B в разделе 6](#6-модули-и-сессии)).

### 🛠 Ещё

| Функция | Описание |
|---------|----------|
| **📍 Новая локация** | Быстрое создание карточки локации |
| **🌍 Кросс-город** | Пометить присутствие персонажа в другом городе |
| **📕 Закрыть хронику** | Архивировать завершённую хронику |
| **🔄 Индекс событий** | Пересобрать `archive/events.md` из всех модулей |
| **📇 Реестр персонажей** | Синхронизировать `characters_index.md` |

### Проверка
- **Проверить ссылки** — найти битые внутренние ссылки в markdown-файлах
- **Автоисправление** — починить автоматически

### 🤖 Модели AI
- Управление ключами провайдеров
- Назначение провайдеров на функции
- Перезапуск сервера

## 16. Поиск

Раздел **Поиск** — полнотекстовый поиск по городу.

Ищет по:
- Именам и описаниям персонажей
- Названиям и описаниям локаций
- Событиям хроник
- Названиям и содержимому модулей

Минимум 3 символа для поиска. Результаты группируются по типу, клик — переход к объекту.

## 17. Панель целостности

Раздел **Панель** содержит блок **Целостность данных** (разворачивается по клику).

### Проверки

- **Битые ссылки** — ссылки в markdown-файлах на несуществующие файлы
- **Сироты** — файлы персонажей/локаций без записи в индексах
- **Дубликаты слагов** — два персонажа с одинаковым `Слаг`
- **Неполные карточки** — отсутствуют обязательные поля

До 40 проблем каждого типа с кнопкой «Показать ещё».

### Проверка канона

Поле «Вставь текст сцены» → ИИ сверяет с фактами из карточек:
- Живы ли персонажи в сцене (статус «Мёртв» / «Летаргия»)
- Соответствуют ли даты состоянию персонажей
- Нет ли фактических противоречий с карточками

## Советы Рассказчику

**Воркфлоу сессии:**

```
До игры:
  Хроники → открытая хроника → + Модуль → заполни концепцию → 🪄 Сгенерировать

Во время игры:
  Открой модуль на телефоне/планшете → вкладка Сценарий

После игры:
  Инструменты → 📓 Сессия → заполни лог → Записать
  Карточки персонажей → Дневники → ✍️ Сгенерировать ИИ

По завершении:
  Страница модуля → 🔒 Закрыть модуль
```

**Нити — твой дашборд незакрытых дел.** Открывай раздел Нити в начале каждой сессии — что тлеет, что горит.

**Граф связей** полезен перед модулем с политическими интригами — видно кто кому обязан и где могут возникнуть неожиданные союзы.

**ИИ-реплики НПС** во время сессии — открой карточку персонажа на телефоне, введи ситуацию и читай ответ голосом НПС.
