# ⚙️ СИСТЕМНОЕ ПРАВИЛО: Добавление и редактирование персонажей

> 📌 **Внимание!** Это правило применяется автоматически при любом запросе на создание, редактирование или добавление персонажа.
> ✅ Сверено со структурой `cities/<город>/`. Город-нейтрально: `<город>` — slug города; `<линейка>` ∈ vampires/fairies/mortals/werewolves/mages/hunters; `<slug>` — ASCII-слаг персонажа (= имя папки). Городская специфика (иерархия, имена-дубли) — `cities/<город>/rules/`.

## 📂 Где хранятся данные

Персонаж — это **папка-слаг** `cities/<город>/characters/<линейка>/<slug>/` со следующим содержимым:

| Путь | Назначение |
|---|---|
| `<slug>/<slug>.md` | карточка персонажа (этот протокол) |
| `<slug>/<slug>-sheet.md` | V20-лист (опц., см. [character_sheet_v20.md](character_sheet_v20.md)) |
| `<slug>/art/` | изображения (`<slug>_NN.ext`) |
| `<slug>/journal/` | дневники (`ГГГГ-ММ.md` + `retrospective.md`, см. [diary_rules.md](diary_rules.md)) |

Линейки (`<линейка>`): `vampires` / `fairies` / `mortals` / `werewolves` / `mages` / `hunters`.

**Сводный справочник города:** `cities/<город>/archive/characters_index.md`. Имя кириллицей берётся из H1; папка — ASCII-слаг.

## 🔁 Протокол добавления НПС

1. **Проверка дубликатов:** Поиск по имени, кличке, внешности, роли, сиру/дитя, домену. Если персонаж уже существует → редактируй существующую запись, не создавай новую.
2. **Выбор папки:** Определи линейку WoD. Создай папку `cities/<город>/characters/<линейка>/<slug>/`, где `<slug>` — ASCII-слаг имени (напр. `Франсуа Вийон` → `fransua_viyon`). Карточка — `<slug>/<slug>.md`. Удобнее всего создавать через `node tools/new_npc.js <город> <линейка> "<Имя>"` — он сразу делает папки, шаблон и строку в индексе.
3. **Использование шаблона:** Вставляй карточку строго по структуре ниже.


## ⚙️ Правила

**Синхронизация связей:** Если указываешь сиру/дитя/союзника, проверь его карточку. Если есть → добавь обратную ссылку в поле «Отношения».

**Обработка пробелов:** Если данные отсутствуют, используй `[неизвестно]` или `⚠️ Требуется уточнение`. Не придумывай лор без явного запроса.

**Мотивация (опционально, но желательно для ключевых НПС):** три строки-поля `- **Хочет:**` / `- **Боится:**` / `- **Рычаг:**` — по одной короткой фразе. При заполнении генерация (сценарии, реплики) считает НПС проактивным: он делает шаги к своей цели в сценах без игроков (контракт — `system/schema/card_schema.md`).

**Промт изображения:** После создания карточки обязательно:
1. Добавить `🎨 Промт для генерации изображения` в карточку (3 блока по структуре из [portret.md](portret.md)).
2. Добавить `## 🖼️ Изображения` в конец карточки (или `⏳ Изображение не предоставлено`).
3. Добавить краткую запись о персонаже в `cities/<город>/archive/characters_index.md` в нужный раздел.

## ⚠️ Ограничения

- Не нарушай иерархию Камарильи/Клана/Двора без явного указания пользователя.
- Не смешивай термины WoD и Changeling без пояснений.
- Имена, даты и топонимы должны соответствовать канону активного города (см. `cities/<город>/city.md` и `cities/<город>/rules/`).
- Не включать «take pose from reference» в промт — поза задаётся при генерации.
- При работе с визуальными референсами сначала обновляй карточку персонажа (секция «Внешность»), затем промт.

## 🚫 Критические запреты — имена-дубли

Городские конфликты имён (когда два разных персонажа носят одно имя) держатся в `cities/<город>/rules/`. Перед генерацией сцены с «спорным» именем — сверься с городским каноном.

> Париж 2010: «Аня» — два разных персонажа (Аня Грос ≠ личность Мэла). Полная таблица различий — [`cities/paris/rules/paris_canon.md`](../../cities/paris/rules/paris_canon.md).
